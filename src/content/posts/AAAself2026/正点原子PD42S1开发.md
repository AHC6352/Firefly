## 关于数据处理
### 端序
STM32为小端系统多，字节数据的低位字节存储在低地址，高位字节存储在高地址
但是该电机为了使电机数据格式符合直觉，采用了大端序输出

### 数据转换

```C
typedef union
{
 float f_val; /* 单精度浮点数 (4 字节) */
 uint32_t u32_val; /* 32 位无符号整数 (4 字节) */
 uint8_t bytes[4]; /* 字节数组，用于观察内存布局 */
} converter_t;

```
该联合体共享内存，如：
定义：converter_t a;

写入a.f_val的数据同时对应a.bytes[4]的数据，
按字节发送数据，即发送a.bytes[4]

由于单片机是小端系统，数据发送时应转换为大端序


### 数据发送

```C
CAN_Driver_SendExtendedData(0x00001000, (uint8_t *)&frame, sizeof(frame), 10);
```
正常是用数组发送数据，也可以直接将数据按顺序写入结构体，然后直接强制类型转换，用指针指向发送数据的内存。同样的需要先进行端序转换的操作

### 数据读取
 
注意在非中断回调读取时开启canRX中断时，会自动进入到can的中断回调函数，但是处理却又不在中断回调处理，这就导致can回调取走了fifo邮箱的数据，回调外处理就会出现读不到数据的情况，所以非回调处理应该在发送指令前关闭中断



发送多读取指令时需要有2ms时间间隔，否则会因为占线而无法正确读取数据
```C
if(cl == 0 && HAL_GetTick() - tick >= 10)
      {
        CAN_StepMotor_ReadSpeed(0x01, &speed_rpm1);
        tick = HAL_GetTick();
        cl = 1;
      }
      if(cl == 1 && HAL_GetTick() - tick >= 2)
      {
        CAN_StepMotor_ReadSpeed(0x02, &speed_rpm2);
        tick = HAL_GetTick();
        cl = 0;
      }
```
这里利用系统时钟保证读取速度数据之间至少间隔2ms，同时又不阻塞程序执行
